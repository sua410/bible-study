# 성경 문제집 웹앱 - 시작 가이드

수아님이 이 폴더에서 받게 되는 파일:
- `index.html` - 웹앱 본체 (브라우저에서 더블클릭하면 바로 동작)
- `manifest.json` - 폰 홈화면에 추가할 때 쓰이는 설정
- `시작가이드.md` - 이 문서

---

## 1단계: 바로 써보기 (설치·계정 필요 없음)

가장 먼저, 동작이 잘 되는지 확인해보세요.

1. `index.html` 파일을 더블클릭하면 브라우저에서 바로 열립니다.
2. **오늘 공부** 탭에서 샘플 22문제(창세기 20개 + 시편/잠언 암송 2개)를 풀어볼 수 있어요.
3. 각 문제 답을 확인한 뒤 **다시 / 어려움 / 보통 / 쉬움** 중 하나를 누르면 다음 출제 간격이 자동으로 계산됩니다.

### 지금 단계에서 가능한 것
- 매일 5-10분 학습 (설정 탭에서 하루 새 문제 개수 조절 가능)
- 자주 틀리는 문제 자동으로 더 자주 출제
- 진도와 통계 확인
- 진도 JSON 파일로 내보내기/가져오기 (수동 백업)

### 아직 불가능한 것 (다음 단계에서 해결)
- 폰과 PC 간 진도 자동 동기화
- 가족·친구와 공유

## 2단계: Firebase로 로그인·동기화 활성화

수아님이 Firebase 콘솔에서 직접 설정해야 하는 단계입니다. **약 15분 소요**되고 모두 무료예요.

### 2-1. Firebase 프로젝트 만들기

1. https://console.firebase.google.com 접속해서 구글 계정으로 로그인
2. **프로젝트 추가** 클릭
3. 프로젝트 이름: `성경문제집` (원하는 이름 아무거나)
4. Google Analytics는 **사용 안 함** 선택 (개인용이라 불필요)
5. **프로젝트 만들기** 클릭, 잠시 기다림

### 2-2. 웹 앱 등록

1. 프로젝트 만들어진 화면에서 **웹 아이콘(`</>` 모양)** 클릭
2. 앱 닉네임: `성경공부 웹` 같이 입력
3. "Firebase Hosting" 체크박스는 **체크하지 않음** (GitHub Pages 쓸 거라서)
4. **앱 등록** 클릭
5. 다음 화면에서 **`firebaseConfig` 객체**를 복사해두세요. 이런 모양이에요:

```javascript
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "성경문제집-xxxxx.firebaseapp.com",
  projectId: "성경문제집-xxxxx",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
```

### 2-3. 구글 로그인 활성화

1. 왼쪽 메뉴에서 **빌드 → Authentication** 클릭
2. **시작하기** 버튼 클릭
3. **Sign-in method** 탭 → **Google** 클릭
4. **사용 설정** 토글을 켜고, 지원 이메일은 본인 구글 이메일 선택
5. **저장**

### 2-4. Firestore 데이터베이스 만들기

1. 왼쪽 메뉴에서 **빌드 → Firestore Database** 클릭
2. **데이터베이스 만들기** 클릭
3. **프로덕션 모드로 시작** 선택 (보안 규칙은 다음 단계에서 설정)
4. 위치는 `asia-northeast3 (서울)` 선택
5. **사용 설정**

### 2-5. 보안 규칙 설정 (자기 데이터만 읽기/쓰기 허용)

1. Firestore Database 화면에서 **규칙** 탭 클릭
2. 아래 코드로 교체:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

3. **게시** 클릭

### 2-6. 승인된 도메인 추가 (GitHub Pages 주소)

GitHub Pages 배포 후 (다음 단계) 다시 와서:
1. Authentication → Settings → **승인된 도메인**
2. **도메인 추가** → `{본인깃허브아이디}.github.io` 입력

### 2-7. index.html에 Firebase 코드 추가

> 이 부분은 수아님이 직접 하기 부담스러우시면, 복사해둔 `firebaseConfig`만 알려주시면 제가 코드를 추가해드릴 수 있어요.

`index.html`을 텍스트 편집기(메모장, VS Code 등)로 열어서 `<head>` 안 가장 아래쪽에 다음을 추가하고, `firebaseConfig` 부분에 2-2단계에서 복사한 값을 붙여넣으세요:

```html
<script type="module">
import { initializeApp } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js";
import { getAuth, signInWithPopup, GoogleAuthProvider, onAuthStateChanged, signOut } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-auth.js";
import { getFirestore, doc, setDoc, getDoc } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js";

const firebaseConfig = {
  // 여기에 2-2단계에서 복사한 firebaseConfig를 붙여넣기
};

const app = initializeApp(firebaseConfig);
window._fbAuth = getAuth(app);
window._fbDb = getFirestore(app);
window._fbProvider = new GoogleAuthProvider();

window.signIn = async function() {
  try {
    const result = await signInWithPopup(window._fbAuth, window._fbProvider);
    window.state.user = { uid: result.user.uid, name: result.user.displayName, email: result.user.email };
    document.getElementById('userInfo').textContent = result.user.displayName;
    // 클라우드에서 진도 불러오기
    const snap = await getDoc(doc(window._fbDb, 'users', result.user.uid));
    if (snap.exists()) {
      window.state.progress = snap.data().progress || {};
      localStorage.setItem("bible_workbook_v1", JSON.stringify(window.state.progress));
    }
    window.renderAuth && window.renderAuth();
  } catch(e) { alert("로그인 실패: " + e.message); }
};

window.signOut = async function() {
  await signOut(window._fbAuth);
  window.state.user = null;
  document.getElementById('userInfo').textContent = '로그아웃 상태';
  window.renderAuth && window.renderAuth();
};

// 진도가 바뀔 때마다 클라우드에 자동 저장 (디바운스)
let syncTimer = null;
const originalSave = window.saveProgress;
window.saveProgress = function() {
  originalSave && originalSave();
  if (!window.state.user) return;
  clearTimeout(syncTimer);
  syncTimer = setTimeout(async () => {
    try {
      await setDoc(doc(window._fbDb, 'users', window.state.user.uid),
        { progress: window.state.progress, updatedAt: new Date().toISOString() });
    } catch(e) { console.error("동기화 실패", e); }
  }, 2000);
};
</script>
```

또한 기존 `index.html` 스크립트 안에서 `state`, `renderAuth`, `saveProgress` 같은 변수들이 외부에서 접근 가능하도록 `window.state = state;` 같은 줄을 추가하면 됩니다 (제가 다음에 도와드릴 수 있어요).

## 3단계: GitHub Pages에 배포하기

폰에서도 접속할 수 있도록 무료 호스팅에 올립니다. **약 10분 소요**.

### 3-1. GitHub 계정 만들기 (이미 있으면 건너뛰기)

1. https://github.com 가입
2. 사용자명 정하기 (예: `sua410`) — 이게 나중에 주소가 됨

### 3-2. 저장소 만들기

1. 우측 상단 **+** → **New repository**
2. Repository name: `bible-study` (또는 원하는 이름)
3. **Public** 선택 (Private은 GitHub Pages 무료 한계 있음)
4. **Create repository**

### 3-3. 파일 업로드

1. 만들어진 저장소 페이지에서 **uploading an existing file** 링크 클릭
2. 이 폴더의 `index.html`, `manifest.json`을 드래그해서 올리기
3. 화면 아래 **Commit changes** 클릭

### 3-4. GitHub Pages 활성화

1. 저장소 상단의 **Settings** 탭 클릭
2. 왼쪽 메뉴에서 **Pages** 클릭
3. **Source** 아래 **Branch**를 `main`, 폴더는 `/ (root)`로 선택
4. **Save** 클릭
5. 1-2분 기다리면 페이지 위쪽에 주소가 나옴:
   `https://{사용자명}.github.io/bible-study/`

### 3-5. 폰에서 홈 화면에 추가

**아이폰 (Safari):**
1. 위 주소에 접속
2. 공유 버튼 (네모+화살표 아이콘) 누르기
3. **홈 화면에 추가**

**안드로이드 (Chrome):**
1. 위 주소에 접속
2. 오른쪽 위 점 3개 메뉴
3. **홈 화면에 추가** 또는 **앱 설치**

## 자주 묻는 질문

**Q. Firebase 비용이 나올 수 있나요?**
무료 한도(매일 읽기 5만 회, 쓰기 2만 회)는 한 사람이 매일 5-10분 공부하는 수준의 1000배 이상이에요. 20명이 써도 무료 한도 안에서 끝납니다.

**Q. 문제를 더 추가하려면?**
`index.html` 파일의 `QUESTIONS = [` 부분에 같은 형식으로 객체를 추가하면 됩니다. 다음 세션에서 더 많은 책을 추가하는 작업을 도와드릴 수 있어요.

**Q. 만약 Firebase 단계가 너무 복잡하면?**
지금 상태(로컬 저장만)로도 한 기기에서는 완전히 잘 동작해요. 폰이나 PC 중 한 곳에서만 쓰셔도 됩니다. 동기화는 정말로 필요해질 때 천천히 추가하셔도 돼요.
